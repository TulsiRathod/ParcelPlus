# Deploying ParcelPlus to Render

This project ships a [`render.yaml`](render.yaml) blueprint that provisions three
things on [Render](https://render.com):

- **parcelplus-db** — managed PostgreSQL
- **parcelplus-backend** — Spring Boot API (built from [BackEnd/Dockerfile](BackEnd/Dockerfile))
- **parcelplus-frontend** — the React app served as a static site

> Heads-up: the frontend and backend need each other's public URLs, which Render
> only assigns after the first deploy. So the flow is **deploy → fill in a few URLs
> → redeploy**. It takes about 10 minutes end to end.

## 0. Prerequisites

1. Push this repository to GitHub (or GitLab).
2. Have a **Google Maps JavaScript API key**. In Google Cloud Console restrict it by
   HTTP referrer once you know the frontend URL (you can add the referrer in step 4).
3. A Render account (the free tier is enough to try it out).

## 1. Create the blueprint

1. Render dashboard → **New + → Blueprint**.
2. Connect the repo. Render detects `render.yaml` and lists the database + two services.
3. Click **Apply**. Render will:
   - create the Postgres database,
   - build and start the backend (the DB credentials are injected automatically, and
     `JWT_SECRET` is generated for you),
   - build the frontend.
4. You'll be prompted for the `sync: false` variables. You can leave the frontend ones
   blank for now (we set them in step 3) and leave backend `CORS_ALLOWED_ORIGINS` blank.

## 2. Note the service URLs

After the first deploy, open each service and copy its URL, e.g.:

- Backend: `https://parcelplus-backend.onrender.com`
- Frontend: `https://parcelplus-frontend.onrender.com`

(The exact names may have a random suffix if the name was already taken.)

## 3. Wire the two services together

**Frontend → backend.** In the **parcelplus-frontend** service → *Environment*, set:

| Key | Value |
|-----|-------|
| `REACT_APP_API_BASE_URL` | `https://parcelplus-backend.onrender.com/api` |
| `REACT_APP_WS_BASE_URL`  | `wss://parcelplus-backend.onrender.com` |
| `REACT_APP_GOOGLE_MAPS_API_KEY` | your Maps key |

> These are baked into the bundle at build time, so after changing them click
> **Manual Deploy → Clear build cache & deploy** to rebuild.

**Backend → frontend (CORS).** In **parcelplus-backend** → *Environment*, set:

| Key | Value |
|-----|-------|
| `CORS_ALLOWED_ORIGINS` | `https://parcelplus-frontend.onrender.com` |

Saving env vars triggers a redeploy. Wait for both to go green.

## 4. Restrict the Maps key

Back in Google Cloud Console, restrict the API key to the frontend referrer
`https://parcelplus-frontend.onrender.com/*`.

## 5. Create an admin user

There's no admin self-signup. Register a normal user through the app, then promote it:

1. Render dashboard → **parcelplus-db** → **Connect** → copy the **PSQL command**, or
   open the built-in shell.
2. Run:
   ```sql
   UPDATE users SET role = 'ADMIN' WHERE email = 'you@example.com';
   ```
3. Log in via the normal client login — you'll be routed to `/admin`.

## 6. Smoke test

- Visit the frontend URL; register and log in.
- Confirm an unauthenticated API call is rejected:
  `curl -i https://parcelplus-backend.onrender.com/api/users/all` → `401/403`.
- Log in as the admin and confirm the dashboard counts load.

## Notes & limitations of the free tier

- **Spin-down:** free web services sleep after ~15 min idle; the first request after
  that takes ~30–60s to wake. Upgrade the backend plan to avoid this.
- **Database expiry:** a free Render Postgres is deleted ~90 days after creation. Use a
  paid plan (or back up) for anything you intend to keep.
- **Schema:** `JPA_DDL_AUTO` is `update` so Hibernate creates/updates tables on first
  boot. Once the schema is stable, switch it to `validate` and manage changes with
  migrations.
- **Secrets:** `JWT_SECRET` is generated by Render and the DB password is managed by
  Render — neither is in the repo. Rotate any credentials that were ever committed.
